# User Flows

## Purpose

Capture the primary customer journeys supported by the project.

## Scope

Public website journeys only.

## Key Flows

### View Clinic Information

1. User opens the website.
2. User reads clinic details, doctor information, services, and contact data.
3. User navigates to booking or contact actions.

### Book an Appointment

1. User opens the booking form.
2. User enters personal and pet details.
3. User submits the request.
4. The website should forward the request to the backend in the production implementation.

### Browse Products

1. User opens the shop section.
2. User filters and searches products.
3. User adds items to the cart.
4. User reviews UPI payment instructions.

### Read Content

1. User opens services, blogs, FAQs, gallery, testimonials, or events.
2. Content is displayed from the configured backend source in the production implementation.
